# Senior UX Designer / Researcher — Common Requirements (2025-2026)

주요 채용 플랫폼의 시니어 UX 디자이너/리서처 JD 종합 분석.
indie-ux 스킬 강화를 위한 참고 자료.

---

## 1. Technical Skills (Hard Skills)

| Category | Common Requirements | Notes |
|----------|-------------------|-------|
| **User Research Methods** | User interviews, contextual inquiry, card sorting, diary studies, surveys, A/B testing, persona creation, journey mapping | 정성+정량 모두 요구. 80% 리서처가 AI 보조 분석 도구 활용 |
| **Wireframing / Prototyping** | Low-fi ~ High-fi prototyping, rapid prototyping, interactive prototype 제작 | Figma Make (AI 기반 프롬프트→와이어프레임) 활용 역량 신규 요구 |
| **Usability Testing** | Moderated/unmoderated testing, task analysis, heuristic evaluation, think-aloud protocol | 정량 지표 (misclick, heatmap, success rate, bounce rate) 해석 능력 필수 |
| **Information Architecture** | Site map, navigation design, content hierarchy, taxonomy, labeling systems | Card sorting + tree testing으로 검증하는 역량 포함 |
| **Design Systems** | Component library 구축/유지, design token, atomic design methodology | Accessibility audit를 디자인 시스템 레벨에서 수행 |
| **Design Tools** | Figma (primary), Sketch, Adobe Creative Suite, InVision, Overflow | Figma가 사실상 표준. AI 플러그인 (QoQo, UX Pilot) 활용 역량 추가 |
| **Research Tools** | Optimal Workshop, UserTesting, Lookback, Lyssna, Dovetail, CoLoop | AI 기반 분석 도구 (자동 전사, 테마 생성, 클립 추출) 경험 우대 |
| **Data Visualization** | Tableau, Google Data Studio (Looker Studio), 리서치 인사이트 시각화 | 비기술 이해관계자 대상 프레젠테이션 역량 포함 |
| **Accessibility** | WCAG 2.1 AA 이상, contrast ratio, focus order, motion sensitivity, semantic structure | 와이어프레임 단계부터 접근성 감사 수행 ("shift-left accessibility") |
| **Interaction Design** | Micro-interactions, motion design, state management, responsive behavior | 인지 부하 최소화 원칙 적용 능력 |

## 2. Senior Differentiators (Mid → Senior Gap)

| Competency | Description | Source |
|-----------|-------------|--------|
| **End-to-End Ownership** | 프로젝트 전체 lifecycle을 주도. 문제 발견→정의→우선순위 제안→해결까지 소유 | UX School, June UX |
| **Strategic Problem Discovery** | 사용자가 인지하지 못하는 잠재 문제와 감정/행동 패턴을 발굴하여 로드맵 우선순위에 반영 | UX School, Quora |
| **Autonomous UX Ownership** | 리서치부터 검증까지 전 사이클을 단독 추진. 멘토링/팀 퍼실리테이션은 팀 환경 한정; 인디에서는 독립 실행이 핵심 | UX Competency Matrix, Xcede |
| **Data-Driven Decision Making** | KPI 달성 경험, A/B 테스트 설계/해석, 정량 지표로 디자인 의사결정 정당화 | June UX, Bonanza Design |
| **Stakeholder Communication** | 디자인 결정을 비즈니스 언어로 설명. 다양한 이해관계자와 효과적으로 협상 | NN/g, ZipRecruiter JD |
| **Cross-Functional Leadership** | 디자인팀 외부와의 협업을 주도. 역할 경계 모호한 상황에서 갈등 해결 | UXPin, NN/g |
| **Design Advocacy** | 조직 전체에 user-centered design 문화 전파. 리서치 결과를 경영진에게 프레젠팅 | GitLab Handbook, Velvet Jobs |
| **Deadline Negotiation** | 현실적 일정 산정 및 협상. 프로젝트 전체 맥락에서 태스크 우선순위 조율 | UX Survival Guide |

## 3. Soft Skills

| Skill | Detail |
|-------|--------|
| **Communication** | 기술/비기술 이해관계자 모두에게 리서치 결과와 디자인 근거를 명확히 전달 |
| **Storytelling** | 데이터와 인사이트를 narrative로 구성하여 공감과 행동 변화를 이끌어내는 능력 |
| **Empathy** | 사용자 관점 내재화. 인터뷰·관찰에서 표면적 발언 너머의 실제 니즈를 포착 |
| **Critical Thinking** | 가정을 의심하고, 다수 접근법을 비교 평가하며, 편향을 인지하고 완화 |
| **Facilitation** | 워크숍, 디자인 스프린트, 브레인스토밍 세션을 효과적으로 설계하고 진행 |
| **Adaptability** | 빠르게 변하는 요구사항, 조직 구조, 도구 환경에 유연하게 대응 |
| **Conflict Resolution** | 디자인 방향성에 대한 이견 조율. PM/Eng 간 트레이드오프 합의점 도출 |

## 4. Experience Requirements

| Condition | Typical Range |
|-----------|--------------|
| **총 경력 (UX Design/Research)** | 5-10년 (대부분 JD에서 5+ 요구, 7+ 선호) |
| **시니어 타이틀 최소 경력** | 3-5년의 독립적 UX 프로젝트 리드 경험 |
| **학력** | Bachelor's in HCI, HCDE, Cognitive Psychology, Design 등. 석사 우대 (필수 아님) |
| **포트폴리오** | 필수. 시각물보다 의사결정 과정(why)과 비즈니스 임팩트 설명 중시 |
| **인증/자격증** | 선택. Google UX Certificate, NN/g 등 — 포트폴리오보다 우선순위 낮음 |
| **산업 경험** | SaaS, B2B, e-commerce, FinTech, HealthTech 등 도메인 경험 우대 |
| **급여 범위 (US)** | $122K-$196K (중간값), Senior Interaction Designer 평균 $205K |

## 5. 2025-2026 Trend Keywords

| Trend | Status |
|-------|--------|
| **AI-Assisted Design Workflow** | 주류 진입. 73% 디자이너가 "AI as design collaborator"를 2026 최대 임팩트로 선택 |
| **AI-Powered UX Research** | 급성장. 80% 리서처가 AI 활용. 자동 전사·테마 생성·감성 분석 도구 확산 |
| **Shift-Left Accessibility** | 표준화 진행. 와이어프레임 단계부터 접근성 감사 수행. EU Accessibility Act 2025 시행 |
| **Cognitive Accessibility** | 부상 중. 인지 다양성 고려한 디자인. 인지 부하·주의력·기억력 배려 |
| **Functional Minimalism** | 확산 중. 장식적 요소 제거, 기능 중심 UI |
| **Calm Technology** | [SPECULATIVE] 부상 중. 사용자를 방해하지 않는 "차분한" 디지털 경험 설계 (철학 수준, JD 요구 사례 드묾) |
| **Design System as Product** | 성숙 단계. 디자인 시스템을 내부 제품처럼 운영 — versioning, adoption metrics 관리 |
| **Design Engineering Convergence** | 가속화. 디자이너의 기술적 이해 (HTML/CSS, React, API) 기대치 상승 |
| **Specialization over Generalism** | 전환기. UX Research, Interaction Design, Accessibility 등 전문 영역 심화 선호 |

## 6. Role Boundary Note

**UX vs Designer**: UX Researcher/Designer는 사용자 행동 조사·인사이트·플로우 설계 담당. Product/UI Designer는 시각적 시스템·브랜드·컴포넌트 소유. 인디 환경에서는 겸임이 일반적이나, UX는 "왜 이렇게 설계하는가"를, Designer는 "어떻게 보여줄 것인가"를 주도.
