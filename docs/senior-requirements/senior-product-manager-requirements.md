# Senior Product Manager — Common Requirements (2025-2026)

주요 채용 플랫폼의 시니어 프로덕트 매니저 JD 종합 분석.
indie-planner 스킬 강화를 위한 참고 자료.

---

## 1. Technical Skills (Hard Skills)

| Category | Common Requirements | Notes |
|----------|-------------------|-------|
| **Product Strategy** | Product Vision 수립, 장기 로드맵 설계, Market Positioning, Go-to-Market (GTM) 전략 | 시니어 JD의 90%+에서 필수 언급 |
| **Roadmapping** | Outcome-driven Roadmap 작성, 분기별 우선순위 설정, 기능 vs 전략 트레이드오프 판단 | 2026년 트렌드: 태스크 기반 → 결과(Outcome) 기반 전환 |
| **User Research** | 정성적/정량적 리서치 설계 및 실행, 사용자 인터뷰, 페르소나 개발, Jobs-to-be-Done 프레임워크 | Discovery 역량 강조 추세 |
| **Data Analysis** | SQL 기본, Product Analytics (Amplitude, Mixpanel, GA4), Funnel/Cohort 분석, A/B Testing 설계 및 해석 | "data-driven decision making" 거의 모든 JD에 포함 |
| **PRD / Spec Writing** | 비즈니스 케이스 정의, 기능 명세, User Story/Acceptance Criteria, Success Metrics 정의 | 명세 명확성(specification clarity) 강조 |
| **Agile / Scrum** | Scrum/Kanban 프로세스 리드, Backlog Refinement, Sprint Planning, Retrospective 주도 | CSPO/CSM 자격증 우대 |
| **AI / ML Literacy** | LLM 동작 원리 이해, AI Feature 기획, Prompt Engineering, AI Ethics 기본 | 2025-2026 "nice-to-have" → "must-have"로 격상 |
| **Metrics & OKRs** | KPI 설정 (Retention, Churn, NPS, MRR/ARR), OKR 프레임워크 운영 | SaaS 기업에서 특히 MRR/Churn 중심 |
| **Tools** | Jira, Confluence, Figma, SQL, Notion, Linear, Amplitude, Looker | 도구 숙련도 자체보다 도구를 활용한 의사결정 능력 중시 |

## 2. Senior Differentiators (Mid → Senior Gap)

| Competency | Description | Source |
|-----------|-------------|--------|
| **Strategic Ownership** | Mid PM은 기능 단위 최적화, Senior PM은 제품 비전 및 장기 방향성 설정 | Lenny's Newsletter, Product School |
| **0→1 Product Launch** | 신규 제품을 처음부터 설계·출시한 경험. 리서치→설계→빌드→런칭 전체 사이클 주도 | LupaHire, CareerFoundry |
| **Autonomous Execution** | 최소한의 감독 하에 독립적 의사결정. "Lead through context, not control" | Product School |
| **Cross-functional Influence** | 직접 권한 없이 Engineering, Design, Data, GTM 팀을 설득·정렬 | Sembly AI, Thiga |
| **Business Impact Framing** | 기능을 비즈니스 임팩트(매출, 리텐션, CAC)로 번역. C-level 대상 커뮤니케이션 | Aakash G |
| **Autonomous Product Ownership** | PM 지원 없이 전체 제품 사이클을 리드. 멘토링/팀 빌딩은 팀 환경 한정; 인디에서는 독립적 결정과 실행이 핵심 | Product Leadership |

## 3. Soft Skills

| Skill | Detail |
|-------|--------|
| **Communication** | 구두·서면 모두 탁월. Google/Amazon/Meta 리더 85%가 "커뮤니케이션 부재"를 제품 실패 1위 원인으로 지목 |
| **Stakeholder Management** | 상충하는 이해관계 조율, 경영진 buy-in 확보, 정기 업데이트를 통한 기대치 관리 |
| **Influence Without Authority** | 직접 보고 라인 없는 팀을 신뢰·공감·정렬로 이끄는 역량 |
| **Strategic Thinking** | 단기 최적화가 아닌 장기 비전 기반 우선순위 판단. 시장 변화 예측 및 대응 |
| **Adaptability** | 빠르게 변하는 시장·기술 환경에서 전략 피벗 능력 |
| **Problem-Solving** | 복잡한 비즈니스/기술 문제를 구조화하고 실행 가능한 단위로 분해 |
| **Empathy** | 사용자·팀원·이해관계자 관점에서 사고. UX 의사결정과 팀 협업 모두에 적용 |
| **Negotiation** | 리소스 배분, 스코프 조정, 일정 협상 등 일상적 트레이드오프 관리 |

## 4. Experience Requirements

| Condition | Typical Range |
|-----------|--------------|
| **Product Management 경력** | 5-8년 (최소 4년, 상위 기업 7년+) |
| **시니어 레벨 재직 기간** | Mid PM 3-5년 경과 후 승진 또는 이직 |
| **학력** | Bachelor's in CS, Business, Engineering, Marketing (필수) |
| **MBA / 석사** | 우대(preferred), 필수 아님. 대기업일수록 선호 |
| **도메인 경험** | SaaS/B2B 또는 해당 산업 2년+ 경험 빈번 요구 |
| **0→1 경험** | 신규 제품 런칭 1회 이상 (시니어 차별화 핵심) |
| **리더십 경험** | 3명+ 크로스펑셔널 팀 리드 경험 |
| **자격증** | CSPO, PSPO, Pragmatic Institute — 우대 사항 |

## 5. 2025-2026 Trend Keywords

| Trend | Status |
|-------|--------|
| **AI/ML Literacy** | 필수 — LLM 이해, AI Feature 기획, Prompt Engineering이 JD 표준 항목으로 편입 |
| **AI Ethics & Responsible AI** | 부상 — EU AI Act 등 규제 강화로 PM JD에 윤리적 AI 감독 책임 명시 증가 |
| **Agentic Workflows** | [SPECULATIVE] 신규 — 단순 GenAI → 자율 에이전트 기반 워크플로우 설계 역량 요구 (일부 AI 퍼스트 기업 한정) |
| **Product-Led Growth (PLG)** | 강화 — Time-to-Value(TTV) 최소화, 셀프서브 온보딩, 프리미엄 전환 최적화 |
| **Outcome-driven Roadmap** | 전환 — Feature 나열식 → 비즈니스 결과(Outcome) 중심 로드맵으로 전환 |
| **AI-Native Product Design** | 신규 — AI를 부가 기능이 아닌 제품 핵심 아키텍처로 설계 |
| **Data Democratization** | 확대 — PM이 직접 SQL/Analytics 도구로 데이터 접근, 데이터팀 의존도 감소 |
| **Continuous Discovery** | 강화 — Teresa Torres식 주간 사용자 접점, 가설 기반 실험 사이클 상시 운영 |
