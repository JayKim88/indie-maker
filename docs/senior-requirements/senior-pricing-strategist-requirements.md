# Pricing / Monetization Strategist — Common Requirements (2025-2026)

주요 채용 플랫폼의 Pricing Strategist / Monetization Manager JD 종합 분석.
indie-monetize 스킬 강화를 위한 참고 자료.

## 1. Technical Skills (Hard Skills)

| Category | Common Requirements | Notes |
|----------|-------------------|-------|
| **Value-Based Pricing** | 고객 가치 기반 가격 설정, WTP(Willingness-to-Pay) 분석, Van Westendorp / Conjoint 분석 | SaaS 프라이싱의 핵심 방법론 |
| **Data Analysis & Modeling** | Excel 고급(피벗, 매크로), SQL, Python/R, BI 도구(Tableau, Looker) | 모든 JD에서 필수 |
| **SaaS Metrics Mastery** | MRR/ARR, ARPU, LTV/CAC, churn rate, expansion revenue, net revenue retention | 비즈니스 모델 이해도 평가 핵심 |
| **Pricing Model Design** | Tiered, usage-based, credit-based, hybrid, freemium 모델 설계 및 A/B 테스트 | 2025-2026 credit-based 모델 급증 |
| **Competitive Pricing Analysis** | 경쟁사 가격 벤치마킹, feature-price matrix, 포지셔닝 맵 | 정기적 모니터링 체계 구축 |
| **Financial Modeling** | Revenue forecasting, margin analysis, scenario planning, sensitivity analysis | Finance 팀 협업 빈번 |
| **Packaging & Bundling** | 기능별 패키징 전략, add-on 설계, 업셀/크로스셀 기회 식별 | 제품팀과 긴밀 협업 |
| **Experimentation** | Pricing A/B 테스트 설계, price elasticity 측정, cohort 분석 | 가설 기반 접근 강조 |

| Condition | Typical Range |

## 5. 2025-2026 Trend Keywords

| Trend | Status |
|-------|--------|
| **Credit-Based Pricing** | 폭발적 성장 — 2024→2025 사이 126% 증가, AI 비용 관리의 핵심 모델 |
| **AI Inference Cost Modeling** | 필수 — AI 기능 제공 시 토큰/inference 비용 기반 마진 관리 능력 |
| **Usage-Based → Hybrid 전환** | 주류 — 순수 seat-based 모델 퇴조, 사용량 연동 하이브리드 모델 부상 |
| **Outcome-Based Pricing** | [SPECULATIVE] 초기 — AI 에이전트의 "작업 완료" 기반 과금 모델 실험 중 (소수 AI 퍼스트 기업 한정) |
| **Price Transparency & Simplicity** | 회귀 — 복잡한 usage-based 모델에 대한 반발로 단순화 요구 증가 |
| **PLG-Native Monetization** | 정착 — Free tier → paid 전환 최적화, paywall 위치 실험 |
| **AI-Assisted Pricing Optimization** | 성장 — AI 도구를 활용한 동적 가격 최적화 및 churn 예측 |

## 6. Role Boundary Note

**Pricing vs Growth**: Pricing/Monetize는 수익 모델 구조·가격 포인트·패키징을 소유. Growth는 해당 가격 구조 안에서 전환율 최적화·채널 실험을 담당. 둘 다 MRR에 영향을 미치지만, Pricing은 단가·구조를 바꾸고, Growth는 전환량을 바꾼다.
