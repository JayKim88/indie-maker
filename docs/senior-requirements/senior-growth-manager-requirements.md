# Senior Growth Manager — Common Requirements (2025-2026)

## 1. Technical Skills (Hard Skills)

| Category | Common Requirements | Notes |
|----------|-------------------|-------|
| **Analytics & Data** | GA4, Amplitude, Mixpanel, Segment 숙련; 코호트/퍼널/리텐션 분석; SQL 쿼리 능력 | 거의 모든 JD에서 "data-driven" 키워드 필수 등장 |
| **A/B Testing & Experimentation** | Optimizely, VWO, LaunchDarkly 등 실험 도구 운용; 주 3-5회 실험 사이클 운영 경험 | 2026 기준 고성과 팀 평균 주 3-5개 실험 실행, 유의미한 승률 15-25% |
| **Acquisition Channels** | Paid (Google/Meta/TikTok Ads), SEO/ASO, Content Marketing, Referral/Viral loops | 채널 다각화 + CAC 최적화 능력 필수; PLG 기업은 self-serve funnel 경험 우대 |
| **Retention & Lifecycle** | Email drip (Resend, Customer.io), push notification, in-app messaging; churn 분석 | Retention이 growth의 핵심 드라이버로 재조명 (2025-2026) |
| **Funnel Optimization & CRO** | 전환율 최적화, 랜딩페이지 A/B 테스트, 온보딩 퍼널 개선; AARRR 프레임워크 | Time-to-value 단축이 PLG 핵심 지표 |
| **Marketing Automation** | HubSpot, Marketo, ActiveCampaign 등 MAP 운용; 워크플로우 자동화 설계 | AI-driven 워크플로우 구축 경험 우대 |
| **AI/ML 활용** | AI 기반 개인화, 예측 분석, 자동 세그멘테이션, LLM 활용 콘텐츠/실험 | Forrester: 예측 분석 도입 팀 효율 25% 향상 (2026) |
| **Pricing & Monetization** | 구독 모델 설계, 가격 실험, freemium-to-paid 전환 최적화 | SaaS 특화 — PLG 기업에서 특히 중시 |
| **Basic Engineering** | HTML/CSS, SQL, Python/JS 스크립팅; API 연동; 트래킹 코드 직접 구현 | "기술을 이해하는 마케터" 포지셔닝; Full-stack growth |

## 2. Senior Differentiators (Mid → Senior Gap)

| Competency | Description | Source |
|-----------|-------------|--------|
| **Revenue Accountability** | 매출 KPI(MRR, ARR, LTV)에 대한 직접 책임; 실험 결과를 매출 임팩트로 환산 | Market Recruitment, Teal HQ |
| **Strategic Vision** | 채널 단위 최적화 → 회사 전체 성장 전략 수립; GTM(Go-to-Market) 설계 | High Alpha, CXL Career Guide |
| **Cross-functional Leadership** | Product, Engineering, Sales, CS 팀과 협업 주도; 부서 간 OKR 정렬 | Wellfound JDs, Teal HQ |
| **Autonomous Execution** | 팀 없이 채널·실험·분석 전 사이클 단독 추진. 멘토링/팀 빌딩은 팀 환경 한정; 인디에서는 독립 실행력이 핵심 | Market Recruitment, Indeed |
| **Full-stack PM Mindset** | 피처 출시 → 시장 반응 → 전환 → 성장까지 End-to-end 소유; Product Sense | Product School, Airtable |
| **Experimentation Framework** | 개별 실험 실행 → 실험 프레임워크/프로세스 자체를 설계하고 스케일링 | ProductLed, RCKT |
| **Budget & Resource Allocation** | 채널별 ROI 기반 예산 배분; CAC payback period 관리 | Indeed, LinkedIn JDs |

## 3. Soft Skills

| Skill | Detail |
|-------|--------|
| **Analytical Thinking** | 숫자가 "무엇"을 보여주는지 + "왜" 그런지를 연결하는 능력 (Data + Context = Insights) |
| **Communication & Storytelling** | 데이터 인사이트를 경영진/비기술 팀에 설득력 있게 전달; 실험 결과 리포팅 |
| **Active Listening** | 고객 피드백, 시장 신호, 팀 인풋을 정확히 수집하고 반영 |
| **High EQ & Collaboration** | 린 조직에서 다양한 이해관계자와 마찰 없이 협업; 갈등 조율 |
| **Bias for Action** | 완벽한 데이터를 기다리지 않고 80% 확신에서 실행; 빠른 의사결정 |
| **Comfort with Ambiguity** | 스타트업 환경의 불확실성 속에서 우선순위 판단 및 피봇 능력 |
| **Creative Problem-Solving** | 제한된 리소스로 비전형적 성장 채널/전술 발굴 (Growth Hacking 본질) |

## 4. Experience Requirements

| Condition | Typical Range |
|-----------|--------------|
| **총 경력** | 5-10년 (B2B SaaS, Tech Startup 환경) |
| **그로스/마케팅 직무 경력** | 최소 5년 이상 (growth marketing, product marketing, growth PM) |
| **스타트업 경험** | 시리즈 A-C 단계 스타트업에서의 실무 경험 강력 우대 |
| **PLG 경험** | Self-serve + Sales-assisted hybrid 퍼널 운영 경험 (2025-2026 필수화 추세) |
| **리더십 경험** | 3-5명 이상 팀 매니징 또는 cross-functional project lead |
| **학력** | 학사 이상 (Marketing, Business, Data Science, CS); MBA 우대하나 필수 아님 |
| **연봉 범위 (US)** | $122K-$193K (Senior Growth Manager); $130K-$160K+ (Senior Growth Hacker) |
| **포트폴리오** | 구체적 성장 지표 달성 사례 (e.g., "CAC 40% 절감", "activation rate 2x") 필수 |

## 5. 2025-2026 Trend Keywords

| Trend | Status |
|-------|--------|
| **AI-Driven Growth** | 핵심 트렌드 — AI 개인화, 예측 분석, 자동 세그멘테이션이 표준 도구화 |
| **Product-Led Growth (PLG)** | PLG + Sales-Assisted 하이브리드 모델이 주류; 순수 PLG만으로는 불충분 |
| **AI Agents as Users** | [SPECULATIVE] PLG의 "사용자"가 사람 → AI 에이전트로 확장; 활성화 지표·과금 모델 재정의 필요 (초기 탐색 단계) |
| **High-Velocity Experimentation** | 주 3-5개 실험이 고성과 팀 기준; 승률보다 학습 속도 중시 |
| **Privacy-First Growth** | GDPR/CCPA 강화로 1st-party data 전략, cookieless 트래킹 필수 |
| **Retention > Acquisition** | CAC 상승으로 리텐션이 growth lever 1순위로 부상 |
| **Full-Stack Growth PM** | Product + Marketing + Data를 넘나드는 "풀스택" 역할 기대치 확대 |
| **Time-to-Value 최적화** | 온보딩 시 "분 단위"로 가치 전달 요구; 긴 셋업 = 이탈 |
| **Community-Led Growth** | 커뮤니티 기반 바이럴 + 레퍼럴이 유료 채널 보완재로 부상 |

## 6. Role Boundary Note

**Growth vs Pricing**: Growth는 채널·퍼널·실험을 통한 사용자 획득·활성화·리텐션 담당. Pricing/Monetize는 수익 모델·가격 구조·결제 전환 최적화 담당. 둘 다 MRR에 영향을 미치지만 레버가 다름.
