# Senior Product/UI Designer — Common Requirements (2025-2026)

주요 채용 플랫폼의 시니어 프로덕트/UI 디자이너 JD 종합 분석.
indie-designer 스킬 강화를 위한 참고 자료.

---

## 1. Technical Skills (Hard Skills)

| Category | Common Requirements | Notes |
|----------|-------------------|-------|
| **Design Tools** | Figma (필수), Adobe Creative Suite (Photoshop, Illustrator) | Figma가 사실상 업계 표준. Sketch는 레거시 프로젝트 한정 — 신규 학습 불필요 |
| **Prototyping** | Figma Prototyping, Framer, Principle, InVision | 고충실도 인터랙티브 프로토타입 제작 능력 필수 |
| **Design Systems** | 컴포넌트 아키텍처, Design Tokens (DTCG 표준), Figma Variables | 시스템 구축·유지보수 경험 필수. JSON export 가능한 토큰 구조 요구 증가 |
| **Visual Design** | Typography, Color Theory, Layout/Grid, Iconography, Brand Identity | 시각적 일관성과 브랜드 가이드라인 적용 능력 |
| **Motion Design** | After Effects, Lottie, Principle, Figma Smart Animate | Micro-interaction 및 UI 애니메이션. 타이밍·리듬·애니메이션 로직 이해 |
| **User Research** | Usability Testing, User Interview, Survey Design, A/B Testing | 리서치 설계→인사이트 도출→디자인 반영 전체 사이클 |
| **Interaction Design** | Information Architecture, User Flow, Wireframing, Responsive Design | 복잡한 인터랙션 패턴 설계 및 문서화 |
| **Frontend 이해** | HTML, CSS, JavaScript 기초, 개발 핸드오프 | 코딩 필수는 아니나 기술적 제약 이해 및 개발자 협업에 필요 |
| **Analytics** | Amplitude, Mixpanel, Google Analytics, Hotjar | 데이터 기반 디자인 의사결정. 정량적 성과 측정 |
| **AI Tools** | AI 기반 디자인 도구 활용 (이미지 생성, 리서치 자동화, 접근성 검수) | 2025-2026 신규 필수 역량으로 부상 |

## 2. Senior Differentiators (Mid → Senior Gap)

| Competency | Description | Source |
|-----------|-------------|--------|
| **Ownership & Influence** | 티켓 실행이 아닌 제품 방향성 제시. 문제 공간 전체를 파악하고 주도적으로 솔루션 도출 | Toptal, GitLab Handbook |
| **Strategic Thinking** | 비즈니스 목표와 UX 목표를 연결. 로드맵 우선순위에 검증된 유저 니즈 반영 | RethinKHQ, ADPList |
| **Systems Thinking** | 개별 화면이 아닌 전체 제품 생태계 관점에서 디자인. 디자인 시스템 전략 수립 | Hatch, Yardstick |
| **Autonomous Design Ownership** | 전체 제품 디자인을 단독 추진; 비전 수립부터 핸드오프까지 소유. 멘토링/채용은 팀 환경 한정 | Workable, Velvet Jobs |
| **Cross-functional Communication** | PM, 엔지니어, 경영진 등 다양한 이해관계자와 디자인 의사결정 근거를 명확히 소통 | ADPList, Quantic |
| **Data-driven Advocacy** | 데이터와 리서치 결과로 디자인 결정을 뒷받침. 명확한 성공 기준 수립 | GitLab Handbook |
| **Business Acumen** | 디자인 스킬뿐 아니라 비즈니스·산업 전반에 대한 이해. 경쟁 전략 수립 참여 | RethinKHQ, Quantic |
| **Storytelling** | 포트폴리오와 프레젠테이션에서 디자인 결정의 "왜"를 설득력 있게 전달 | Supercharge Design |

## 3. Soft Skills

| Skill | Detail |
|-------|--------|
| **Communication** | 리더십 프레젠테이션, 디자인 비평, 크로스펑셔널 협업에서의 명확한 의사 전달 |
| **Collaboration** | PM, 엔지니어, 리서처, 마케터와의 긴밀한 협업. 리모트 환경 포함 |
| **Critical Thinking** | 가정 검증, 다양한 솔루션 탐색, 트레이드오프 분석 |
| **Adaptability** | 빠르게 변화하는 요구사항과 기술 트렌드에 유연하게 대응 |
| **Empathy** | 사용자 중심 사고. 다양한 사용자 그룹의 니즈와 제약 이해 |
| **Curiosity** | 새로운 도구·방법론·트렌드에 대한 자발적 학습과 실험 |
| **Conflict Resolution** | 이해관계자 간 상충하는 요구사항 조율 및 합의 도출 |

## 4. Experience Requirements

| Condition | Typical Range |
|-----------|--------------|
| **경력 연수** | 5-8년 (일반), 8+년 (모션/시스템 특화 포지션) |
| **학력** | 학사 이상 — HCI, Graphic Design, Interaction Design, Fine Art 등 (동등 실무 경력 대체 가능) |
| **포트폴리오** | 필수. 실제 출시 제품 기반 케이스 스터디 3-5개. 프로세스·방법론·결과 문서화 |
| **디자인 시스템 경험** | 대규모 제품에서 시스템 구축 또는 운영 경험 (Senior 필수 요건으로 격상) |
| **리더십 경험** | 주니어 멘토링, 디자인 리뷰 주도, 채용 면접 참여 경험 |
| **도메인 경험** | SaaS, Fintech, E-commerce, Health 등 해당 산업 경험 우대 |
| **급여 (US)** | $120K-$205K base (지역·회사 규모에 따라 편차). FAANG/Big Tech $200K-$300K+ (RSU 포함) |

## 5. 2025-2026 Trend Keywords

| Trend | Status |
|-------|--------|
| **AI-augmented Design Workflow** | 필수 — AI 도구를 일상 워크플로에 통합 (이미지 생성, 리서치 합성, 접근성 검수) |
| **Design Tokens (DTCG Standard)** | 급부상 — JSON 기반 토큰 표준화 필수. AI 워크플로 및 멀티플랫폼 호환을 위한 기반 |
| **Accessibility-first Design** | 필수 — WCAG 준수가 법적 요건으로 강화. AI 기반 접근성 자동 검수 도구 활용 증가 |
| **Design-to-Code Alignment** | 부상 — Figma Variables → 코드 바인딩 직결. Dev Mode 활용, 디자인-개발 간극 최소화 |
| **AI-native Design Systems** | [SPECULATIVE] 신규 — AI가 소비할 수 있는 구조로 디자인 시스템 재설계 (개념 단계, 실제 채용 요구 사례 드묾) |
| **Voice UI / Conversational Design** | 성장 — 음성 인터페이스, 챗봇 UX 등 대화형 디자인 전문성 수요 증가 |
| **Specialization over Generalism** | 전환 — "풀스택 디자이너"에서 전문 분야 심화로 이동 |

## 6. Role Boundary Note

**Designer vs UX**: Designer는 시각적 일관성·브랜드·컴포넌트 시스템 소유. UX Researcher는 사용자 행동 조사·검증·인사이트 도출 담당. 인디 환경에서는 겸임이 일반적이나 역할 우선순위는 구분 필요.
