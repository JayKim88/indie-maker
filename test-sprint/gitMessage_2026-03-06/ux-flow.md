# UX Flow: GitMessage

> Created: 2026-03-06 | Phase 1.5 UX Sprint

---

## Mental Model

**Target user tools**: VS Code Source Control panel, GitHub Desktop, Terminal
**Dominant UI pattern**: SCM input box (top of Source Control panel) + icon buttons
**Design decisions carried forward**:
- Button adjacent to SCM message input box (user already looks there)
- Sparkle (✨) icon = AI-assist signal (Copilot standard)
- Inline message fill — no separate panel

**UX Assumption #1**: Target user regularly uses VS Code Source Control panel (not terminal-only git).

---

## Task Flows

### Core Task: Generate commit message
Step 1 → Stage files (git add)
Step 2 → Click ✨ button in SCM panel
Step 3 → Message auto-filled ✅ **(Activation Event)**

**3 steps — Target met**

### Convention config
Step 1 → Open Settings (gear icon / command palette)
Step 2 → Select format (Conventional / gitmoji / custom)
Step 3 → Saved ✅

### Pro upgrade
Step 1 → Status bar shows "47/50 uses"
Step 2 → Click status bar → Upgrade webview opens
Step 3 → Stripe checkout ✅

---

## Navigation Architecture

**Primary navigation**: VS Code native (command palette, SCM panel, status bar)
No custom navigation — extension integrates into existing VS Code chrome.

**Recommended**: Do not add sidebar views or activity bar icons for MVP.
Reason: cognitive load — users should not need to learn a new panel.

---

## Onboarding Strategy

**Strategy B: Zero-friction**
- Install → ✨ button appears in SCM panel immediately
- No tutorial, no welcome screen
- First use = generate button click
- Aha Moment: seeing generated message for the first time

Rationale: Developer audience — any friction = churn.
