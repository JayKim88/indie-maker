## Core User Scenario (1 scenario)

**JTBD**: When I finish coding a feature and am about to commit, I want a meaningful commit message generated automatically from my diff, so I can maintain a clean git history without context-switching.

**User**: Solo developer / indie hacker using VS Code or Cursor
**Trigger situation**: After staging changes in Source Control panel
**Goal**: Get a conventionally formatted commit message in 2 seconds
**Success criteria**: MVP is complete when the user can click the generate button, review the message, and commit — without leaving VS Code
